NMAP Basics

Network scanning fundamentals and examples

This post was inspired by Network Chuck’s YouTube video “Nmap Tutorial to Find Network Vulnerabilities”

Finding your IP Addresses on a connected networks

First, I’ll need to show all network interfaces and their addresses, a command is needed which shows these network interfaces.

ip addr show

This input is basic and beneficial, it gives

  • All network interfaces (eg VPNs and Bluetooth on my device)
  • Whether network interfaces are up or down
  • MAC addresses
  • Subnet info

Looks complicated right? Lets break it down and gather key info from this output.

First of all, loopback represents an internal self connection address; the computer talking to itself.

In the second line, enp0s25 simply refers to the ethernet — the state being DOWN means there is no cable plugged in.

The fourth line is the most important for what we’re doing; wlp3s0 refers to Wifi

link/ether 10:0b:a9:3f:44:f8 is my MAC address for this device and...

inet 192.168.0.156/24 is my device’s IP address while /24 represents the subnet mask meaning my local network (my WIFI) is actually 192.168.0.0/24.

Think of a subnet mask as the fence around a network:
/24 draws a line from where devices belong. For example, /24 represents everyone whose IP starts with 192.168.0 being in the local network, supporting up to 254 devices.

The smaller the number, the more devices that can be supported on a local network; /16 = ~65k devices, /8 = millions.

Therefore, I now know that 192.168.0.0/24 is my local network. Time to use nmap.

For Ubuntu/Debian linux devices, use the command

sudo apt update && sudo apt install -y nmap

Nmap is now installed.

Now, I want to run a TCP connect scan on ports 80 and 443 across every host in the subnet 192.168.0.0/24 which, as previously mentioned, represents my entire network.

Ports are numbered endpoints on a device that let different programs talk over the network. Each assigned port ensures traffic goes to the right program.

Examples:
  • Port 80 (HTTP): Handles unencrypted web traffic.
  • Port 443 (HTTPS): Handles encrypted web traffic.
  • Port 25 (SMTP): Used to send emails between mail servers.

Now we know what ports are, lets scan them on the network

sudo nmap -sT -p 80,443 192.168.0.0/24

lets break this code down;

  • sudo: run as root — not strictly required here but it prevents permission-related errors
  • nmap: the tool we’re using to scan the networks
  • -sT: a TCP connect scan (a full TCP handshake)
  • -p 80,443: scan only these ports
  • 192.168.0.0/24: scan the subnet
Output (examples taken from a larger result):

This is a basic TCP connection. The handshake is loud and detectable by SIEMs and can trigger alarms.

Some devices likely to have port 80 and 443 open are routers (such as Sky in the example) because they provide web admin dashboards. Filtered means a firewall is blocking the handshake request.

Smart TVs, printers, and streaming boxes may also expose web dashboards — these are common targets because they often have default or weak credentials.

Some bots automatically scan for port 80 due to its lack of encryption; they may try default passwords or attempt to fingerprint services.
